Sass projects can use `salt-theme-gen` in two ways: the CSS variable approach (recommended for most projects) or generating a `_tokens.scss` partial with native Sass variables. Both work; the choice depends on whether your team prefers `var(--color-primary)` or `$light-primary`.

## Approach 1 — CSS variables (recommended)

Generate a `theme.css` file and import it in Sass. This is the simpler approach and gives you dark mode for free.

```scss
// src/main.scss
@import './theme.css';  // or <link> it in <head>

body {
  background-color: var(--color-background);
  color:            var(--color-text);
  font-size:        var(--text-md);
  font-family:      system-ui, sans-serif;
}

.btn-primary {
  background:    var(--color-primary);
  color:         var(--color-on-primary);
  border-radius: var(--radius-md);
  padding:       var(--space-sm) var(--space-lg);

  &:hover   { background: var(--state-primary-hover); }
  &:active  { background: var(--state-primary-pressed); }
  &:focus-visible {
    outline: 2px solid var(--state-primary-focused);
    outline-offset: 2px;
  }
}
```

CSS variables are first-class in Sass — you can use `var()` anywhere. Dark mode works via `:root[data-theme="dark"]` in the generated CSS.

## Approach 2 — Generated `_tokens.scss` partial

For teams that want `$primary` syntax, generate a Sass partial at build time.

**Generation script** (`scripts/generate-sass-tokens.mjs`):

```js
import { generateTheme } from 'salt-theme-gen';
import { writeFileSync, mkdirSync } from 'node:fs';

const theme = generateTheme({ preset: 'ocean' });

const kebab = s => s.replace(/([A-Z])/g, '-$1').toLowerCase();

function colorsBlock(mode, prefix) {
  return Object.entries(mode.colors)
    .map(([k, v]) => `$${prefix}-${kebab(k)}: ${v};`)
    .join('\n');
}

function scalesBlock(mode) {
  const spacing  = Object.entries(mode.spacing).map(([k, v]) => `$spacing-${k}: ${v}px;`).join('\n');
  const radius   = Object.entries(mode.radius).map(([k, v]) => `$radius-${k}: ${v}px;`).join('\n');
  const fonts    = Object.entries(mode.fontSizes).map(([k, v]) => `$font-${k}: ${v}px;`).join('\n');
  return [spacing, radius, fonts].join('\n\n');
}

// Sass map for iteration
function colorsMap(mode, name) {
  const entries = Object.entries(mode.colors)
    .map(([k, v]) => `  '${kebab(k)}': ${v}`)
    .join(',\n');
  return `$${name}: (\n${entries}\n);`;
}

const sass = [
  '// Generated by salt-theme-gen — do not edit manually',
  '// Run: node scripts/generate-sass-tokens.mjs\n',
  '// Light mode',
  colorsBlock(theme.light, 'light'),
  '\n// Dark mode',
  colorsBlock(theme.dark,  'dark'),
  '\n// Scales',
  scalesBlock(theme.light),
  '\n// Sass maps',
  colorsMap(theme.light, 'light-colors'),
  colorsMap(theme.dark,  'dark-colors'),
].join('\n');

mkdirSync('src/styles', { recursive: true });
writeFileSync('src/styles/_tokens.scss', sass, 'utf8');
console.log('✓ src/styles/_tokens.scss written');
```

Add to `package.json`:

```json
{
  "scripts": {
    "build:tokens": "node scripts/generate-sass-tokens.mjs",
    "build": "npm run build:tokens && vite build"
  }
}
```

## Using generated Sass variables

```scss
// src/styles/main.scss
@use './tokens' as t;

body {
  background-color: t.$light-background;
  color:            t.$light-text;
  font-size:        t.$font-md;
}

.btn-primary {
  background:    t.$light-primary;
  color:         t.$light-on-primary;
  border-radius: t.$radius-md;
  padding:       t.$spacing-sm t.$spacing-lg;

  &:hover { background: t.$light-state-primary-hover; }
}
```

## Dark mode with Sass variables

```scss
// Bridge pattern: use Sass vars to populate CSS custom properties
:root {
  --color-background: #{t.$light-background};
  --color-primary:    #{t.$light-primary};
  --color-text:       #{t.$light-text};
}

[data-theme='dark'] {
  --color-background: #{t.$dark-background};
  --color-primary:    #{t.$dark-primary};
  --color-text:       #{t.$dark-text};
}

body {
  background-color: var(--color-background);
  color:            var(--color-text);
}
```

This gets the best of both worlds: Sass variables as the generation source of truth, CSS variables for runtime switching.

## Sass maps for utility generation

```scss
@use './tokens' as t;

// Generate .text-primary, .text-danger, etc.
@each $name, $value in t.$light-colors {
  .text-#{$name} { color: $value; }
  .bg-#{$name}   { background-color: $value; }
}

// Generate .p-xs, .p-sm, .p-md etc.
$spacing-map: (
  'xs': t.$spacing-xs, 'sm': t.$spacing-sm,
  'md': t.$spacing-md, 'lg': t.$spacing-lg,
);

@each $name, $value in $spacing-map {
  .p-#{$name}  { padding: $value; }
  .px-#{$name} { padding-left: $value; padding-right: $value; }
}
```

## Auto-inject via Vite

To make `_tokens.scss` available in every file without `@use`:

```ts
// vite.config.ts
export default {
  css: {
    preprocessorOptions: {
      scss: {
        additionalData: `@use '/src/styles/tokens' as t;`,
      },
    },
  },
};
```

Every `.scss` file now has `t.$light-primary`, `t.$spacing-md`, etc. without any import.

## Which approach to use

Use **CSS variables** if: you want runtime dark mode switching, you work across multiple frameworks, or your team is comfortable with `var()`.

Use **Sass variables** if: your team works heavily in Sass and prefers `$` syntax, or you need Sass maps for utility class generation.

Full guide: [learn.esalt.net/salt-theme-gen/integrations/sass/](https://learn.esalt.net/salt-theme-gen/integrations/sass/)
